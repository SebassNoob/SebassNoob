Discord bot


An overview

For the uninitiated, Discord is organised into servers with a common topic (eg. gaming servers, servers where one can learn a language, servers for friends to hang out) where a user can join multiple servers. Bots can be added to servers where they perform or automante a variety of tasks, including user moderation and text based games. The particular bot I created, annoybot, has a goal of boosting interactivity withih servers by allowing server members to 'annoy' other server members in a harmless way.
Creation and operation process

Using a package called Discord.py, I could interact with discord's API through sending requests through this module. By typing commands prefixed with a '$', end users could make the bot send messages into 'channels' in a server. Some of these messages could range from insults to jokes to interactive button-based games. An example of what the bot can do. (1.8.0a)
Development on this started in June 2021, where I had a month to kill. Initial progess was slow, as the package was not beginner friendly. I experimented with the features the module brought and made the core features that the bot has today. Eventually, after production release, the bot grew slowly to 100 servers. Due to discord's tyranny new security policy introduced in late 2020, I had to verify my bot with my personal details, in order to allow the bot to grow past 100 servers. As of June 2022, 1 year after development has begun, the bot sits in 510 servers.
Limitations and challenges

Backend limitations
Replit is designed to only run code for a maximum loop of 30 minutes while the user is offline. Once 30 minutes has elapsed, the process will automatically terminate. This affects my bot in that I have to continuously ping the server with an external application called Uptimerobot which sends GET requests every 5 minutes to a linked web page on the server my bot is running on. This ensures that the page is always online and therefore the bot is, too.
Discord's ratelimits on its API
In order to prevent API abuse, Discord has automatic ratelimits on all API requests coming from a source. If too many requests are made in a short amount of time, Discord will automatically block incoming requests for a short period of time. Therefore, the only way to reduce the number of API calls per unit time is to push it to the end user as well. Every command that the end user uses will have a predetermined cooldown time in order to prevent spam and therefore prevent too many API calls being made in a short amount of time. Although this system is in place, the bot still may get ratelimited and go offline since it has to accommodate requests coming from 500 servers, which is no small feat.
